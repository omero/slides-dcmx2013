Que es un tema ?



En Drupal lo definiremos de una manera coloquial
como la apariencia final de nuestro site.

Un tema es una colección de archivos que definen
la capa de presentación del sitio



Entonces que es un tema base?



Un principio fundamental un punto de partida
para realizar nuestro poderoso tema y de esta
forma evitar armar un tema diferente por cada
proyecto perdiendo nuestro valioso tiempo y esfuerzo.

Como funciona un tema?



Es una serie de overrides ...


En el Core de Drupal el sistema de temas 
y el motor de plantillas nos proporcionan
 de la infraestructura para sobrescribir 
cualquier capa del tema.
El tema en drupal es el todopoderoso, 
tiene la ultima palabra y css 
que se muestra en el navegador

Que tipos de tema base

existen?


Diremos que la mayor parte de los temas
residen en 3 categorias:

  • Adaptados al nucleo de Drupal
  • Frameworks ajenos adaptados a Drupal
  • Componentes anexos en un mega tema

Adaptados a Drupal


Temas donde los desarrolladores se esfuerzan por mejorar el margen de beneficio del nucleo de Drupal y uso de variables para mejorar

  • Flujo de Trabajo
  • Pureza Semantica
  • Normas de Cumplimiento

Algunos Ejemplos: Zen, Boron, Mothership

Diremos que este tipo de adaptación es como una remodelación de una casa

Frameworks ajenos adaptados a Drupal


Frameworks de desarrollo web muy comunes 
hoy en dia, para un facil desarrollo 
adaptados a los temas


Algunos ejemplos: NineSixty, SquareGrid, BluePrint

Diremos que este tipo de adaptación es como 
comprar una casa nueva.

Componentes anexos a un mega tema


Estos temas hacen : "TODO"

  • Extienden el Core Drupal con modulos adicionales
  • Adaptan el marco de Drupal (p/e añaden HTML5)
  • Adoptan grids CSS


Algunos ejemplos: AT, Fusion, Omega

Diremos que este tipo de adaptación es como 
una mega residencia.

Observaciones






Tenemos un par de cosas interesantes


El Core es flexible: Al permitir a nosotros los desarrolladores para adapater cualquier marco en la red lo cual nos permite hacer un porting asia cualquier nueva tendencia en la web

El Core se queda corto: Los temas anexos nos muestran la forma en que se han superado los temas con los modulos que se han realizado para extender la funcionalidad del core.

Características de algunos temas


Mencionaremos diferentes temas y que tienen para ofrecernos

ZEN


Zen es un tema Sass/Compass-powered, gestionable con drush responsivo y hecho en HTML5

Todos sus componentes se relacionan con el buen rendimineto markup HTML5, ligero.

Falta de inexperiencia , Si no sabes css es complicado realizar un subtheme

Detamo


Es un tema base separado en 3 grids separados. Grids fluido desktop (960px) tablet (640px) y movil (320px).

Es facil usar detamo para realizar un tema responsivo basdo en 960

El control responsivo esta en el archivo template no necesitamos hacer el control css en el layout

Ninesixty


Implementación en Drupal del Sistema Grid 960 un framework CSS

Es ligero y Flexible

Es un Framework CSS se puede apilar con otros marcos o temas básicos solo necesitamos copiar el CSS

Square Grid


Un tema base ligero basado en el framework Square Grid para rapido y ligero theming

Ligero = Pocas opciones configurables

No es un tema que intenta hacer el café es muy minimo

Adaptative Theme

Responsivo , HTML5, Soporte a Panels, Panelizer y DS usa SASS, normalize.

Herramientas para móvil: responsive JS, Responsive Layout integración con Panels & DS, Extensas Opciones de Tema

Opciones de configuración para el movimiento de bloques en el responsive

Integración con drush \m/

Omega Theme


Tema base para desarrollo de temas moviles amigables

Adaptación para actuales y futuros Dispositivos

Uno de los temas que mas contribuciones tiene por la comunidad

Buenas Alianzas con Context y Delta :P

Integración con Drush \m/
